Key Architectural Invariants

Epistemic Enforcement: No default-constructible EpistemicMetadata. Every concept MUST have explicit type + trust. Knowledge is NEVER deleted, only INVALIDATED.
READ-ONLY Boundaries: CognitiveDynamics, UnderstandingLayer, MiniLLMs -- all READ-ONLY on LTM. Only SystemOrchestrator + IngestionPipeline write to LTM.
HYPOTHESIS Enforcement: All MiniLLM outputs are HYPOTHESIS. No automatic FACT promotion. THEORY-to-FACT requires human review.
Bounded Values: Activation in [0,1], Trust in [0,1], Salience in [0,1], Weight in [0,1]. All clamped.
Lock Hierarchy: LTM (1) > STM (2) > Registry (3) > Embeddings (4). Never acquire lower-numbered lock while holding higher-numbered lock.
Tools Not Agents: All subsystems are mechanical delegates. No subsystem makes autonomous epistemic decisions.

Class: StringPool

Append-only mmap-backed string storage. 4GB limit (uint32_t offsets). No deduplication. No compaction.

WAL System

WALOpType: STORE_CONCEPT(1), ADD_RELATION(2), REMOVE_RELATION(3), INVALIDATE_CONCEPT(4), UPDATE_METADATA(5).

WALEntryHeader (32 bytes): magic "WL19", sequence_number, operation, payload_size, CRC32 checksum.

CRC32: FIPS polynomial 0xEDB88320. Lazy-initialized 256-entry lookup table.

Class: WALWriter

Append-only with atomic writes via writev(2). fsync after every write. Sequence recovery on open.

Class: WALRecovery

Replays WAL entries after crash. IDEMPOTENT: skips entries already applied. Stops on first corrupt entry.

Class: PersistentLTM CORE

mmap-backed LTM. Same API as LongTermMemory but with crash recovery.

Write pattern: WAL log BEFORE mmap write. In-memory indices rebuilt on load.

Checkpoint System

CheckpointManager: Creates tagged checkpoints. Saves STM snapshots + store sync.

CheckpointRestore: Restores STM from checkpoint. Validates data integrity.

Invariant: WAL entries logged before mmap write. Recovery is idempotent. next_id never decreases. File always has valid header. Records contiguous after header.
Weak Point: No concurrent access (single-threaded). WAL grows unbounded until checkpoint. 4GB StringPool limit. No record-level locking. CRC32 is weak vs cryptographic hash.

Module: concurrent

Files: concurrent/shared_ltm.hpp, shared_stm.hpp, shared_registry.hpp, shared_embeddings.hpp, lock_hierarchy.hpp, deadlock_detector.hpp

Purpose: Thread-safe wrappers for all shared subsystems. Lock hierarchy enforcement.

Lock Hierarchy

LTM (1) > STM (2) > Registry (3) > Embeddings (4). Never acquire lower-numbered lock while holding higher-numbered. Debug builds throw on violation.

Class: SharedLTM READ-WRITE

shared_mutex wrapper. Multiple readers, single writer. All write ops use unique_lock, reads use shared_lock.

Class: SharedSTM READ-WRITE

Per-context fine-grained locking. Global shared_mutex + per-context shared_mutex (stored as unique_ptr to prevent rehash invalidation). ContextMutexRef RAII pattern holds both locks.

Class: SharedRegistry READ-WRITE

Registry-level + per-model locking. ModelGuard RAII pattern: holds registry shared_lock + per-model exclusive lock.

Class: SharedEmbeddings READ-WRITE

shared_mutex with fast-path/slow-path optimization for lazy context creation. Callback-based mutable access.

Class: HierarchicalMutex

RAII wrapper enforcing lock hierarchy. check_hierarchy() in debug builds, no-op in release.

Class: DeadlockDetector (DEBUG only)

Singleton. Logs lock events (max 10000). DFS cycle detection in wait-for graph. No-op stub in release.

Invariant: Lock ordering: LTM(1) before STM(2) before Registry(3) before Embeddings(4). All SharedLTM/STM/Registry/Embeddings operations hold appropriate locks. Per-context mutexes stored as unique_ptr (rehash-safe).
Weak Point: No timeout on lock acquisition. DeadlockDetector drops oldest events on overflow. No automatic deadlock resolution.

Key Architectural Invariants

Epistemic Enforcement: No default-constructible EpistemicMetadata. Every concept MUST have explicit type + trust. Knowledge is NEVER deleted, only INVALIDATED.
READ-ONLY Boundaries: CognitiveDynamics, UnderstandingLayer, MiniLLMs — all READ-ONLY on LTM. Only SystemOrchestrator + IngestionPipeline write to LTM.
HYPOTHESIS Enforcement: All MiniLLM outputs are HYPOTHESIS. No automatic FACT promotion. EpistemicBridge maps KAN training results to trust scores.
Bounded Values: Activation ∈ [0,1], Trust ∈ [0,1], Salience ∈ [0,1], Weight ∈ [0,1]. All clamped.

Key Design Decisions

  • No default EpistemicMetadata: EpistemicMetadata() = delete — forces explicit epistemic decisions at every knowledge creation point
  • Knowledge never deleted: INVALIDATED status with trust < 0.2 preserves epistemic history
  • MicroModel per concept: Each concept has its own 430-parameter bilinear model (W·c+b, 10D) for personalized relevance scoring
  • KAN for function learning: B-spline basis functions (Cox-de Boor recursion) enable smooth univariate approximation without fixed activation functions
  • Bidirectional LLM↔KAN refinement: Hypotheses from LLM → KAN training → residual feedback → refined hypothesis
  • Epistemic Bridge: Maps KAN training metrics (MSE, convergence) to trust scores: MSE<0.01→THEORY, MSE<0.1→HYPOTHESIS, else→SPECULATION
  • Parallel thinking streams: Category-based (Perception/Reasoning/Memory/Creative) with priority scheduling and anti-starvation
  • Lock-free work queues: Vyukov bounded MPMC queue for stream task distribution
  • WAL + Checkpoints: Write-ahead log for durability, periodic checkpoints for fast recovery
  • Foundation seeding: 4-tier bootstrap (~233 concepts, ~144 relations) provides ontological scaffold for new knowledge

Known Weak Points & Gaps

⚠️ MiniLLM→LTM validation loop not wired: Proposals from UnderstandingLayer go through KanValidator but the SystemOrchestrator doesn't yet write validated results back to LTM (~50 lines missing)
⚠️ MicroModels learn topology not semantics: Training is one-shot from graph structure. No incremental learning from query feedback.
⚠️ InteractionLayer doesn't exist: MicroModels don't communicate with each other. Only passive RelevanceMap aggregation, no Hopfield-like activation propagation between models.
⚠️ EMBED_DIM = 10 too small: Stability analysis recommends 16D minimum. Current 10D limits representational capacity.
⚠️ No weight normalization: System unstable at node degree >5 without normalization of inter-model weights.
⚠️ No SIGINT handler in main(): Ctrl+C in command mode skips shutdown() and final checkpoint.
⚠️ Ollama dependency: ChatInterface and OllamaMiniLLM require external Ollama server. No fallback for offline operation beyond StubMiniLLM.
⚠️ No symmetry enforcement: W_ij inter-model weights (future InteractionLayer) must be symmetric for convergence guarantee.
